# 🏠 LOBBY & MATCHMAKING SPECIFICATION

## Главная страница, выбор режима и комнаты

---

# 1. 📌 Назначение

Модуль отвечает за:

* выбор режима игры (бот / мультиплеер)
* создание комнаты
* отображение списка активных комнат
* присоединение к комнате

---

# 2. 🧱 Архитектура

Client (Next.js UI)
↓ tRPC
Server (matchmaking router)
↓
Database (matches, players)

Принцип:

* сервер хранит состояние комнат
* клиент только отображает и отправляет действия

---

# 3. 🖥 Главная страница (/)

## 3.1 Общий layout

Экран делится на 2 зоны:

LEFT (основная зона):

* выбор режима
* создание комнаты

RIGHT (sidebar):

* список активных комнат

---

# 4. 🎮 Выбор режима

## UI элементы

* Кнопка: "Играть с ботом"
* Кнопка: "Многопользовательская игра"

---

## Поведение

### Играть с ботом

* создаётся матч
* добавляется бот
* пользователь сразу попадает в игру

### Многопользовательская игра

* отображается блок создания комнаты

---

# 5. 🏗 Создание комнаты

## UI

После выбора мультиплеера:

* Заголовок: "Создать комнату"
* Выбор количества игроков:

  * 2 игрока
  * 3 игрока
* Кнопка: "Создать"

---

## Поведение

1. Пользователь выбирает размер (2 или 3)
2. Нажимает "Создать"
3. Клиент вызывает API
4. Сервер:

   * создаёт Match
   * устанавливает maxPlayers
   * добавляет создателя как Player
5. Редирект в комнату (/match/:id)

---

# 6. 🧩 Модель данных (расширение)

## Match

```
Match {
  id: string
  status: "waiting" | "playing" | "finished"
  maxPlayers: number (2 | 3)
  createdAt: Date
}
```

## Player

```
Player {
  id: string
  matchId: string
  userId: string
}
```

# 7. 📡 Список комнат (Sidebar)

## 7.1 Отображение

Каждая комната отображается как карточка:

```
[Room #ID]
Players: 1 / 2
[Join Button]
```

---

## 7.2 Состояния комнаты

### 1. Есть свободные места

* отображается кнопка "Присоединиться"
* карточка активна

### 2. Комната заполнена

* карточка серого цвета
* текст: "Нет места"
* кнопка отсутствует или disabled

---

## 7.3 UI состояния

```
RoomCard:
- id
- playersCount
- maxPlayers
- isFull
```

# 8. 🔌 API (tRPC)

## getRooms

```
input: {}
output: Room[]
```

## createRoom

```
input: { maxPlayers: 2 | 3 }
output: { matchId }
```

## joinRoom

```
input: { matchId }
output: { success }
```

# 9. 🔄 Realtime обновление

## События

* ROOM_CREATED
* ROOM_UPDATED
* PLAYER_JOINED

---

## Поведение

* список комнат обновляется в realtime
* при заполнении комнаты → меняется состояние UI

# 10. 🎯 UX сценарии

## Сценарий 1: создание комнаты

1. Пользователь выбирает мультиплеер
2. Создаёт комнату на 2 игрока
3. Видит себя как 1/2
4. Ждёт второго игрока

---

## Сценарий 2: подключение

1. Пользователь видит список комнат
2. Нажимает "Присоединиться"
3. Попадает в комнату

---

## Сценарий 3: комната заполнена

1. Пользователь видит комнату 2/2
2. Карточка серого цвета
3. Текст: "Нет места"

---

# 11. ⚠️ Ограничения

* максимум 3 игрока в комнате
* нельзя присоединиться к полной комнате
* пользователь не может быть в двух комнатах одновременно

---
